Lemons CMake API

CMake options

  • LEMONS_AAX_SDK_PATH: Path to the AAX SDK.
  • LEMONS_BUILD_DOCS: Build the Lemons documentation.
  • LEMONS_BUILD_TEMPLATES: Build the Lemons project templates.
  • LEMONS_BUILD_TESTS: Build the Lemons unit tests.
  • LEMONS_BUILD_TEST_LAB: Build the TestLab application.
  • LEMONS_CREATE_AGGREGATE_TARGETS: Generate ALL_APPS and ALL_PLUGINS targets, populated accordingly. Defaults to ON.
  • LEMONS_ENABLE_INTEGRATIONS: Enable all available integrations by default. Defaults to ON.
  • LEMONS_IGNORE_ALL_EXTRAS: Ignore everything except what's needed to make compiling your project work. Defaults to OFF.
  • LEMONS_IOS_SIMULATOR: Build for an iOS simulator, rather than a real device. Defaults to ON.
  • LEMONS_JUCE_BRANCH: The branch of the JUCE GitHub repository to use - should be develop or master. Defaults to develop.
  • LEMONS_MAC_SDK_VERSION: Version of the MacOS SDK to use. Defaults to 10.13.
  • LEMONS_MAC_UNIVERSAL_BINARY: Builds for x86_64 and arm64. Defaults to ON.
  • LEMONS_PLUGINVAL_LEVEL: Pluginval testing level. Defaults to 10.
  • LEMONS_SANITIZERS_TO_ENABLE: List of sanitizers to enable.
  • LEMONS_VST2_SDK_PATH: Path to the VST2 SDK.

CMake modules

Lemons is designed to be as modular as possible; its CMake features, integrations, and the JUCE modules themselves (and even JUCE itself) are all included by projects via CMake module includes. For example, a basic plugin's CMakeLists.txt might look something like this:

cmake_minimum_required (VERSION 3.19)
project (NewPlugin VERSION 0.0.1 LANGUAGES CXX)
add_subdirectory (GetLemons)
include (LemonsPluginModules)
juce_add_plugin (...)
lemons_configure_juce_plugin (TARGET ${PROJECT_NAME} ASSET_FOLDER assets TRANSLATIONS)

It is possible to use Lemons only for its default CMake set-up, without including any of the JUCE modules in the build tree at all. Categories of JUCE modules can be included by the relevant module in the 'Juce module inclusion' category below:
